PHASE 0 — Setup Kolay
Geliştirme ortamını sıfırdan kuruyoruz. Boş bir klasörden başlayıp React, TypeScript ve Tailwind ile çalışan bir iskelet oluşturuyoruz. Bu phase bittiğinde tarayıcıda açılan, "Hello CV Builder" yazan bir sayfan olacak.

Hedef: Çalışan bir React + TypeScript + Tailwind iskeleti.

  • npm create vite@latest client ile React + TS projesi Vite ile boş bir React + TypeScript projesi oluştur. Hızlı dev server ve modern build aracı.
  • Tailwind CSS kurulumu ve config Utility-first CSS framework. bg-blue-500 gibi hazır class'larla styling yapacağız.
  • shadcn/ui init + Button, Input gibi temel componentler Hazır, accessible component kütüphanesi. Sıfırdan Button yazmaya gerek kalmaz.
  • ESLint + Prettier kurulumu Kod kalitesi (ESLint) ve otomatik formatlama (Prettier). Standart bir yazım şekli sağlar.
  • Klasör yapısı oluştur (components, pages, store, types, lib, hooks) Projenin organize klasör hiyerarşisi. Her dosyanın yeri belli olur.
  • Git init ve ilk commit Versiyon kontrolü başlat. Her phase sonunda commit atacağız.
PHASE 1 — Types ve Layout Kolay
Projenin "iskeleti"ni oluşturuyoruz. CV verisinin TypeScript'te nasıl görüneceğini tanımlıyoruz, sayfalar arası geçişi sağlıyoruz, ana ekranların yerleşimini hazırlıyoruz. Henüz form yok ama header, sidebar, sayfa düzeni hazır.

Hedef: Sayfalar arası gezilebilen, layout'u hazır site.

  • TypeScript types: Resume, Basics, Work, Education, Skill, Project, Language, Certificate, Volunteer, Publication, CustomSection CV verisinin TypeScript şablonları. Sabit bölümler (deneyim, sertifika, gönüllülük, yayın) + kullanıcının tanımlayacağı özel bölümler için CustomSection type'ı. Basics type'ında profiles array'i (LinkedIn, GitHub, Website).
  • React Router kurulumu Sayfalar arası geçiş yapan navigation kütüphanesi. URL'ler ile component'ler eşleşir.
  • Sayfalar: Home, Builder, Templates, Dashboard, NotFound Sitedeki ana sayfaları boş componentler olarak oluştur. Her birinin URL'i olacak.
  • Layout componentleri: Header, Sidebar, Footer Tüm sayfalarda ortak kullanılacak parçalar. Logo, menü, alt bilgi.
  • Builder sayfası 3 sütun layout: Sidebar | Forms | Preview Editor sayfasının ana düzeni: solda menü, ortada form, sağda canlı önizleme.
  • Mobile responsive (Tailwind breakpoints) Telefonda, tablette, masaüstünde uygun görünüm. Tailwind'in sm: md: lg: prefix'leri ile.
PHASE 2 — Form Components Orta
Kullanıcının CV bilgilerini gireceği form ekranlarını tasarlıyoruz. Her bölüm için ayrı bir form. Henüz girilen veri kaydedilmiyor, sadece input'lar ve görsel tasarım var.

Hedef: Statik ama görsel olarak tamam form componentleri.

  • PersonalInfoForm (name, email, phone, label, summary) Kişisel bilgi formu: isim, iletişim, kısa özet. CV'nin en üst kısmı için.
  • ExperienceForm (company, position, dates, highlights) İş deneyimi formu: şirket, pozisyon, tarihler, başarılar. CV'nin en önemli kısmı.
  • EducationForm (institution, degree, field, dates) Eğitim formu: okul, derece, alan ve tarihler.
  • SkillsForm (tag-style input, level) Yetenek listesi. Tag tarzı (Enter'a basınca chip oluşur) input yapısı.
  • ProjectsForm (name, description, url) Proje formu: yaptığın projelerin adı, açıklaması ve linki.
  • LanguagesForm (name, proficiency) Dil bilgisi formu: dil ve seviye (A1, B2 veya başlangıç, akıcı vs.).
  • Sidebar'dan form'lar arası geçiş Sol menüden tıklayınca o form ana alanda açılsın.
  • PhotoUpload component (PersonalInfoForm içinde) Profil fotoğrafı yükleme alanı: dosya seç butonu, sürükle-bırak desteği, yüklenen fotoğrafın küçük önizlemesi. Kabul edilen formatlar: JPG, PNG, WebP. Max boyut kontrolü.
  • SocialLinksInput (PersonalInfoForm içinde) LinkedIn, GitHub, Website ve diğer profil linkleri. "Yeni link ekle" butonu ile dinamik alan ekleme. Her satırda platform seçici (dropdown) + URL input'u.
  • CertificatesForm (name, issuer, date, url) Sertifika ve kurs formu: sertifika adı, veren kurum, tarih ve doğrulama linki.
  • VolunteerForm (organization, role, dates, summary) Gönüllülük formu: kuruluş, rol, tarihler ve yapılan işin kısa açıklaması.
  • PublicationsForm (name, publisher, date, url) Yayın formu: makale/yazı adı, yayıncı, tarih ve erişim linki.
PHASE 3 — State Management + Live Preview Orta
Projenin sihirli noktası. Zustand ile tüm CV verisini global bir "kasada" tutuyoruz. Form'a yazdığın anda sağ taraftaki preview'da CV'nin nasıl göründüğünü görüyorsun. Sayfayı kapatsan da veri kaybolmuyor.

Hedef: Form'a yazınca preview'da anında görünmesi.

  • Zustand kurulumu: npm install zustand Modern state management kütüphanesi. Redux'tan çok daha basit.
  • resumeStore.ts oluştur (state + actions) CV verisini tutan ve değiştiren merkezi "kasa". Form bu kasaya yazar, preview bu kasadan okur.
  • Persist middleware ile localStorage'a otomatik yazma Zustand'ın persist eklentisi. Veri tarayıcıda otomatik kaydedilir, sayfa yenilense kaybolmaz.
  • Tüm formları store'a bağla (onChange → action) Form input'unu yazınca store'daki ilgili field güncellenir.
  • ResumePreview component yaz Store'u dinleyen ve CV önizlemesini gösteren component. Sürekli güncel kalır.
  • Fotoğrafı store'da tut ve preview'da göster Yüklenen fotoğrafı Base64 string olarak Zustand store'da sakla. localStorage'a persist edilir. Preview template'inde fotoğrafı göster (yuvarlak veya kare çerçeve).
  • sampleResume.ts — örnek CV verisi + "Örnek CV ile Başla" butonu Boş formla başlamak yerine hazır örnek veriyle dolu bir CV sunma seçeneği. Kullanıcı ilk açılışta "Boş Başla" veya "Örnek ile Başla" seçer. Formlarda nasıl doldurulacağını gösterir.
  • İlk template: ClassicTemplate (HTML + Tailwind, A4 görünüm) İlk CV tasarımı. A4 boyutunda, klasik düzenli, profesyonel görünüm.
  • Builder sayfasında preview'ı sağ panele yerleştir Sol formlar, sağda canlı CV. Yazdığın anda sağda değişiyor.
  • Vitest + React Testing Library kurulumu Test altyapısı: vitest, @testing-library/react, @testing-library/jest-dom. İlk test config'i.
  • Store unit testleri (resumeStore.test.ts) updateBasics() çağrınca state doğru değişiyor mu? Persist middleware localStorage'a yazıyor mu? Saf fonksiyon testi — en kolay giriş noktası.
PHASE 4 — Form Validation Orta
Form'ların "akıllı" olmasını sağlıyoruz. Boş alan, geçersiz email, yanlış telefon — bunları tespit edip uyarı veriyoruz. react-hook-form ve Zod kütüphaneleriyle.

Hedef: Hata mesajları ve kontrollerle güvenli formlar.

  • Kur: react-hook-form zod @hookform/resolvers Form yönetimi (RHF) + schema validation (Zod) + ikisini birleştiren resolver.
  • schemas/ klasöründe Zod schema'ları yaz Her formun validation kurallarını TypeScript'le tanımla. "İsim zorunlu, email format kontrolü" gibi.
  • Tüm formları react-hook-form ile refactor et Eski state-bazlı form'ları RHF'in register ve handleSubmit sistemine geçir.
  • Required, email, phone validation Boş alan kontrolü, geçerli email format kontrolü, telefon format kontrolü.
  • Hata mesajları input altında göster Yanlış girişte input'un hemen altında kırmızı bir hata mesajı çıksın.
  • Zod schema testleri (schemas/*.test.ts) Geçerli veri → parse başarılı mı? Boş isim → doğru hata mesajı mı? Geçersiz email → yakalanıyor mu? Pure function testi — input ver, output kontrol et.
PHASE 5 — Dinamik Bölümler + Drag & Drop Orta
Sabit alanlardan dinamik listelere geçiyoruz. Kullanıcı istediği kadar deneyim, eğitim, proje ekleyebilecek ve fareyle sürükleyerek sıralayabilecek.

Hedef: İstediğin kadar item ekleme + sürükleyerek sıralama.

  • useFieldArray ile dinamik listeler (Experience, Education, vs.) react-hook-form'un dinamik liste hook'u. Kaç tane item olacağını runtime'da belirler.
  • Add / Remove butonları "Yeni Deneyim Ekle" ve "Sil" butonları. Liste büyüyüp küçülebilsin.
  • Kur: @dnd-kit/core @dnd-kit/sortable Modern, accessible drag & drop kütüphanesi. React 18 uyumlu.
  • Drag & drop ile bölüm sıralama Item'ları fareyle tutup yer değiştir. "En yeni iş üstte olsun" gibi.
  • Sıralama değişiklikleri Zustand store'a yansısın Yeni sıralama store'a kaydedilsin ki preview'da da güncel görünsün.
  • Component testleri (React Testing Library) Form renderlanıyor mu? "Ekle" butonuna basınca yeni item oluşuyor mu? "Sil" butonuna basınca item kalkıyor mu? Kullanıcı etkileşimlerini simüle eden testler.
  • CustomSectionBuilder — kullanıcı tanımlı özel bölümler "Özel Bölüm Ekle" butonu ile kullanıcı bölüm adını yazar (ör: "Hobiler", "Ödüller"), istediği kadar alan ekler. Phase 5'in dinamik liste altyapısını kullanır. Store'da customSections array'inde tutulur.
PHASE 6 — Çoklu Template + Tema Orta
Tek template'ten 3 farklı tasarıma çıkıyoruz: Classic, Modern, Creative. Kullanıcı bunlar arasında geçiş yapabilecek, ayrıca renk paleti ve font seçebilecek.

Hedef: 3 template + renk/font customization.

  • ModernTemplate ve CreativeTemplate componentleri 2 yeni CV tasarımı: Modern (sade, minimalist) ve Creative (renkli, sıra dışı).
  • TemplateSelector component (kullanıcı template seçer) Template seçim arayüzü. Kullanıcı thumbnail'lara tıklayıp seçer.
  • Store'da templateId field'ı Hangi template'in seçili olduğunu store'da tut. Preview ona göre render etsin.
  • ThemePicker: renk paleti, font, spacing Tema customization paneli. Renk seç, font seç, satır aralığı ayarla.
  • CSS variables ile dynamic theming CSS variable'lar (--primary-color) ile renk değişikliği anında uygulansın.
PHASE 7 — PDF Export + JSON Import/Export Zor
Asıl iş geliyor. Kullanıcı CV'sini gerçek bir PDF dosyası olarak indirebilecek. Ayrıca veri JSON olarak yedeklenip başka bir cihazda geri yüklenebilecek.

Hedef: PDF indirme + JSON yedek alma/geri yükleme.

  • Kur: @react-pdf/renderer PDF üretim kütüphanesi. Kendi component sistemi var (Document, Page, View, Text).
  • PDFDocument component (Document, Page, View, Text) PDF için ayrı bir component yaz. HTML değil, react-pdf'in özel etiketleri.
  • Mevcut template'lerin PDF versiyonları Her template için PDF karşılığını yaz. Görsel olarak önizlemeyle aynı olsun.
  • PDFExportButton (BlobProvider veya PDFDownloadLink) PDF indirme butonu. Tıklayınca dosya kullanıcının bilgisayarına iner.
  • JSON Export: store → .json indirme Tüm CV verisini JSON dosyası olarak indirme. Yedek almak için.
  • JSON Import: file picker → parse → store'a yükle JSON dosyasını yükleyip store'a aktarma. Başka cihazda devam edebilmek için.
  • ATS uyumlu PDF çıktısı ATS (Applicant Tracking System) sistemlerinin CV'yi doğru parse edebilmesi için: seçilebilir metin (görsel değil), basit ve düz yapı, standart bölüm başlıkları ("Experience", "Education"), tablo/grafik yerine düz metin akışı. PDF'in makine tarafından okunabilirliğini test et.
PHASE 8 — Backend Setup Zor
Frontend bitti, sunucu (backend) tarafına geçiyoruz. PostgreSQL veritabanını kuruyoruz, Express.js ile REST API yazıyoruz, Prisma ORM ile DB işlemlerini yönetiyoruz. Henüz frontend'e bağlanmadı — Postman ile test edeceğiz.

Hedef: Çalışan bir REST API + PostgreSQL veritabanı.

  • PostgreSQL local kurulum (Windows installer veya Docker) Bilgisayara veritabanı sistemi kur. pgAdmin ile yönetebilirsin.
  • server/ klasörü, Express + TypeScript setup Backend projesini başlat. Frontend'den ayrı bir klasör.
  • tsconfig.json, entry point src/server.ts TypeScript ayarları ve server'ın giriş dosyası. npm run dev bunu çalıştırır.
  • Prisma init + schema.prisma ile Resume modeli ORM kurulumu ve veritabanı modelinin tanımlanması. Resume tablosu nasıl olacak?
  • npx prisma migrate dev ile DB'ye uygula Tanımladığın schema'yı PostgreSQL'e gerçek tablo olarak oluşturur.
  • Routes: GET, POST, PUT, DELETE /api/resumes CRUD endpoint'leri. CV oluştur, listele, güncelle, sil işlemleri.
  • Controller + Service katmanları İstek yönetimi (controller) ve iş mantığı (service) ayrı katmanlarda. Temiz mimari.
  • Middleware: CORS, validation (Zod), error handler CORS (frontend'in erişimi), Zod (gelen veriyi doğrula), error handler (hata yönetimi).
  • Fotoğraf upload endpoint'i (Multer) POST /api/resumes/:id/photo — Multer middleware ile dosya alımı, boyut/format kontrolü, diske veya cloud storage'a kayıt. Dosya URL'ini DB'ye yaz.
  • Postman / Thunder Client ile tüm endpoint'leri test et API'yi manuel test et. CV oluştur, listele, güncelle, sil — hepsi çalışmalı.
  • Supertest kurulumu + API testleri (*.test.ts) POST /api/resumes → 201 + doğru body? GET → liste geliyor mu? Geçersiz veri → 400 + hata mesajı? Manuel Postman testlerini otomatik koda çevir.
PHASE 9 — Frontend ↔ Backend Entegrasyon Zor
Frontend ve backend ayrı çalışıyordu, şimdi onları birbirine bağlıyoruz. Otomatik kayıt, multi-resume, dashboard. Login olmadığı için anonim UUID sistemi kullanıyoruz.

Hedef: Otomatik kayıt + multi-resume desteği.

  • Frontend'de Axios kurulumu + base URL HTTP client kütüphanesi. Tüm istekler için ortak base URL ve interceptor.
  • lib/api/resumes.ts API client layer API çağrılarını tek bir dosyada topla. Componentler bu fonksiyonları çağırsın.
  • userStore.ts — anonymous UUID üret ve sakla İlk girişte tarayıcıya anonim UUID ata. localStorage'da kalıcı tutulur.
  • Tüm isteklere X-Owner-Uuid header ekle Her API isteğinin header'ında UUID gönder. Backend kimi aradığını anlasın.
  • Backend'de bu header'ı oku, ownerUuid olarak kullan Backend gelen UUID'yi okur ve sadece o kullanıcıya ait CV'leri döner.
  • useAutoSave hook (2sn debounced) Otomatik kayıt. Kullanıcı 2 saniye yazmazsa veri otomatik backend'e gider.
  • Loading / "Saved" indicator Sağ üstte "Kaydediliyor..." veya "Kaydedildi" yazısı. Kullanıcıya feedback.
  • Fotoğraf upload'ı backend'e bağla Base64 yerine gerçek dosya upload'ına geç. Frontend'den FormData ile gönder, backend URL dönsün, store'da URL tutulsun.
  • Loading skeleton'ları (Dashboard + Builder) API'den veri gelene kadar içerik yerine gri placeholder şekiller göster. Dashboard'da CV kartları için skeleton, Builder'da form verisi yüklenirken skeleton. Profesyonel UX ve algılanan hızı artırır.
  • Dashboard sayfası — kullanıcının tüm CV'leri Kullanıcının tüm CV'lerinin listelendiği sayfa. Card view, edit, delete butonları.
  • Multi-resume desteği Kullanıcı birden fazla CV oluşturup aralarında geçiş yapabilsin.
  • Integration testleri (frontend ↔ backend) Form doldur → API'ye gidiyor mu? Dashboard'da CV listesi doğru geliyor mu? Auto-save çalışıyor mu? Uçtan uca akış testleri.
PHASE 10 — Public Share + Deployment Zor
Son aşama. CV'yi link ile paylaşma özelliği ve siteyi internete deploy etme. Bu phase bittiğinde gerçek bir web uygulaman olacak — birine link gönderebileceksin.

Hedef: Paylaşılabilir link + canlı deployment.

  • Backend: POST /api/resumes/:id/share — shareSlug üret Paylaşım için unique bir slug üret (örn: "abc123"). DB'ye kaydet.
  • Public endpoint: GET /api/share/:slug (UUID gerektirmez) Public erişim noktası. Kimse UUID olmadan erişebilir.
  • Frontend SharedResume sayfası (read-only görünüm) Sadece okunabilir paylaşım sayfası. Edit yok, sadece görüntüleme + PDF indir.
  • Share butonu + link kopyalama UI "Paylaş" butonuna basınca link gözüksün ve "Kopyala" butonu olsun.
  • SEO meta tags, 404 page, error boundary SEO için meta tag'ler, bulunamayan sayfa için 404, hatalar için error boundary.
  • Production build (frontend + backend) Üretim için optimize edilmiş build. npm run build ile.
  • Frontend deploy: Vercel veya Netlify Frontend'i internete yükle. Ücretsiz, otomatik deploy, CDN.
  • Backend deploy: Railway veya Render Backend'i (Express server) cloud'a yükle. Ücretsiz tier'ları yeterli.
  • DB hosting: Neon, Supabase veya Railway PostgreSQL Veritabanını cloud'a taşı. Local PostgreSQL artık gerek yok.
  • Environment variables ve custom domain (opsiyonel) Production için .env değişkenleri ve istersen kendi domain'ini bağla.
