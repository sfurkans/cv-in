cv-builder/ (6 dosya)
  • README.mdProje tanıtımı — özellikler, stack, kurulum adımları, env tablosu, proje yapısı, komutlar.
  • CHANGELOG.mdPhase bazlı değişiklik günlüğü (Keep a Changelog formatı). Versiyon yerine Phase 0 → 12 + polish.
  • TODO.mdBacklog: 25 madde, 5 kategori (güvenlik, fonksiyonel, kod kalitesi, tip/veri, UX). Yeni iş yapınca buradaki maddeyi işaretlersin.

  • TANITIM.htmlProje tanıtım sayfası (ürün özellikleri + ekran görüntüleri odaklı).
  • .gitignoreGit'in izlemeyeceği dosyalar (node_modules, dist, .env, uploads, .claude).

Frontend — client/

React 19 + Vite + TypeScript + Tailwind v4 + shadcn/ui + Zustand tabanlı SPA. Tüm UI, state yönetimi, form'lar, CV şablonları, PDF üretimi burada.

client/ (11 config) kök-level ayarlar
  • package.jsonFrontend bağımlılıkları (React 19, Vite 8, Tailwind 4, Zustand, react-hook-form, Zod, react-pdf, dnd-kit, axios, Sentry, lucide) + npm script'leri.
  • tsconfig.jsonKök TypeScript config — tsconfig.app.json ve tsconfig.node.json'a referans verir.
  • tsconfig.app.jsonUygulama kodu için TS ayarları (strict, @/* path alias, ES2022 target, React JSX).
  • tsconfig.node.jsonVite config dosyaları için ayrı TS config (Node env).
  • vite.config.tsVite kurulumu: React plugin, Tailwind plugin, @/ → ./src alias, Vitest setup.
  • eslint.config.jsESLint flat config — TypeScript + React Hooks + Prettier entegrasyonu.
  • .prettierrcPrettier format kuralları (semi false, single quote, 2 space).
  • components.jsonshadcn/ui config — component oluştururken nereye yazılacağını söyler (components/ui).
  • index.htmlSPA giriş HTML. Meta tags (SEO, Open Graph, Twitter Card), favicon, <div id="root"> + main.tsx script.
  • .env.exampleEnv şablonu: VITE_API_URL (default localhost:4000/api), VITE_SENTRY_DSN (opsiyonel).
  • .gitignoreClient'a özel ignore — node_modules, dist, .env local.

  • fonts/NotoSans-Regular.ttfPDF export için Noto Sans normal — Türkçe karakterleri (ş, ğ, ü, ö, ç, ı) destekler.

client/src/ (3 giriş dosyası) uygulama bootstrap'i
  • main.tsxGerçek giriş noktası. React DOM render, StrictMode, ErrorBoundary sargısı ve Sentry.init() (DSN varsa) burada. Uygulama buradan başlar.
  • App.tsxReact Router kurulumu. Builder, Dashboard, Templates sayfaları React.lazy ile ayrı chunk'larda (Home sayfa yükü 2.2MB → 232KB). useAuthBootstrap ile session restore.
  • index.cssTailwind v4 import + shadcn color tokens + :root ve .dark CSS değişkenleri (primary=violet, accent=teal) + bg-brand-gradient ve text-brand-gradient utility'leri.
client/src/components/ (56 dosya) tüm UI bileşenleri

Kök seviye

  • ErrorBoundary.tsxClass component. Tüm uygulamayı sarar, uncaught React hatalarını yakalar, Sentry.captureException'a bildirir, kullanıcıya "Tekrar Dene / Ana Sayfa" fallback UI gösterir.
  • OfflineBanner.tsxInternet bağlantısı koptuğunda ekranın üstünde "Offline" banner'ı gösterir. useOnlineStatus hook'unu dinler.
  • ResumeMiniPreview.tsxDashboard kartlarında gösterilen küçük CV önizlemesi. A4 template'i scale ile thumbnail boyuta indirip iframe'sız render eder.
  • Toaster.tsxtoast store'unu dinleyip ekranın sağ altında mesaj kartları gösterir. Auto-dismiss + icon (success/error/info).

auth/ — kimlik doğrulama UI'sı

  • AuthModal.tsxTablı modal (Giriş / Kayıt). ESC tuşu + dışına tıklayınca kapanır, body scroll lock.
  • AuthModal.test.tsx test9 integration testi — tab değiştirme, başarılı register/login, şifre uyuşmazlığı, API hataları.
  • GlobalAuthModal.tsxLayout seviyesinde mount edilir. authModalStore'u dinleyip modal açık mı kapalı mı karar verir — böylece herhangi bir component openAuthModal() çağırarak modal'ı tetikleyebilir.
  • LoginForm.tsxGiriş formu: react-hook-form + Zod. authStore.login çağırır, başarıda onSuccess callback'i tetikler (örn: otomatik kaydet).
  • RegisterForm.tsxKayıt formu — email, şifre, şifre tekrarı. Min 8 karakter kontrolü, eşleşme kontrolü client'ta.

brand/ — marka görselleri

  • Logo.tsxCv-İn logosu — iki export: LogoMark (sadece ikon, Header'da kullanılır) ve kelime mark.

builder/ — CV düzenleme ekranı

  • BuilderMobileNav.tsxMobile'de section chip'leri (yatay kaydırılabilir nav). Desktop sidebar'ın mobil karşılığı.
  • BuilderSidebar.tsxDesktop sol sidebar. Section'ları listeler, @dnd-kit ile sürükle-bırak sıralama, her section için hata sayısı + tamamlık yüzdesi progress bar.
  • FormPanel.tsxOrta panel. activeSection prop'una göre ilgili form component'ini render eder (dispatcher).
  • PDFExportButton.tsxreact-pdf PDFDownloadLink ile PDF indirme butonu. Template'e göre PDF_TEMPLATES'ten component seçer, dosya adını CV sahibinin isminden üretir.
  • PreviewPanel.tsxSağ panel. 794×1123px (A4 @ 96dpi) canvas, container genişliğine göre scale hesaplar. Üstte SyncStatusIndicator + SaveButton + PDFExportButton.
  • SaveButton.tsxManuel kaydet butonu. Auth yoksa AuthModal açar, başarı sonrası otomatik kaydet callback'i ile devam eder.
  • SyncStatusIndicator.tsx"Kaydedildi / Kaydediliyor / Hata" etiket + relatif zaman ("3 saniye önce").

builder/design/ — tasarım paneli

  • DesignPanel.tsx"Tasarım" tab'ının konteyner'ı. İçinde TemplateSelector, ThemePicker, JsonImportExport bulunur.
  • TemplateSelector.tsx9 template'i grid halinde gösterir (TemplateThumbnail kullanır). Tıklayınca setTemplateId çağırır.
  • ThemePicker.tsxRenk paleti preset'leri, native color picker, fontFamily (sans/serif/mono), spacing (compact/normal/relaxed).
  • JsonImportExport.tsxCV'yi JSON olarak indir + JSON dosyasından yükle. Yüklerken Zod ile validate eder.

builder/dnd/ — sürükle-bırak helper'ları

  • SortableItem.tsx@dnd-kit useSortable sarmalı. İçine verilen children'ı sürüklenebilir yapar, drag handle'ı ayrı prop.
  • SortableList.tsxDndContext + SortableContext sarmalı. Liste için drag-end event'ini yönetir, onReorder callback'i çağırır.

builder/forms/ — CV form'ları (12 bölüm)

  • ProjectsForm.tsxProje listesi (ad, açıklama, URL, tarihler).
  • LanguagesForm.tsxDil + yeterlilik seviyesi (A1/A2/B1/B2/C1/C2 veya başlangıç/orta/akıcı).
  • CertificatesForm.tsxSertifika (ad, veren kurum, tarih, doğrulama URL'si).
  • VolunteerForm.tsxGönüllülük (kuruluş, rol, tarihler, özet).

  • CustomSectionsForm.tsxKullanıcı tanımlı bölümler. Başlık + istediği kadar label/value field'ı (Hobiler, Ödüller, Referanslar için kullanılabilir).

  • SocialLinksInput.tsxLinkedIn/GitHub/Website vb. için dinamik input listesi. Platform dropdown + URL alanı.

server/src/ (17 dosya) tüm backend kodu

Giriş noktaları

  • server.tsEntry point. createApp()'i çağırır, env.PORT'ta listen eder. Prod ve dev'de tsx src/server.ts ile çalışır.
  • app.tsExpress factory. Middleware zinciri: CORS (dev'de tüm localhost portları) → JSON parser (5MB limit) → morgan logger → /uploads static → Sentry init (DSN varsa) → router'lar (/health, /api/auth, /api/resumes) → 404 handler → Sentry error handler → error handler.

config/ — environment

  • config/env.tsZod ile env validation. NODE_ENV, PORT (default 4000), DATABASE_URL, CORS_ORIGIN, JWT_SECRET (min 32), SENTRY_DSN (opsiyonel). Invalid env'de process.exit(1).

controllers/ — HTTP handler'lar

  • controllers/authController.tsregister (201 + user + token), login (200 + user + token), me (200 + user). authService'i sarar, response'u {data} envelope'ına alır.
  • controllers/resumeController.tslist, get, create, update, remove, uploadPhoto. req.userId'yi requireAuth'tan alır, resumeService'i çağırır.

services/ — iş mantığı

  • services/authService.tsregister (email unique check + bcrypt hash + JWT sign), login (password verify + JWT sign), me (user by id). Hataları HttpError ile fırlatır.
  • services/resumeService.tsPrisma CRUD + ownership check. getById owner değilse 403 atar. Diğer method'lar getById'yi çağırarak yetki kontrolü yapar.

routes/ — Express router'ları

  • routes/auth.tsPOST /register + /login (validateBody), GET /me (requireAuth).
  • routes/health.tsGET /health → {status: "ok", timestamp}. Deployment monitoring için.
  • routes/resumes.tsTüm route'lar requireAuth altında. GET /, POST /, GET/PUT/DELETE /:id, POST /:id/photo (multer).

middleware/ — Express middleware'leri

  • middleware/error.tsHttpError class'ı + notFoundHandler + errorHandler. ZodError'ları 400 ValidationError'a, Multer hatalarını UploadError'a, Prisma P2002'yi 409'a, P2025'i 404'e çevirir. Diğerleri 500.
  • middleware/requireAuth.tsAuthorization: Bearer <jwt> header'ını doğrular, req.userId set eder. Geçersizse 401. Global Express type augmentation burada.
  • middleware/upload.tsMulter disk storage — uploads/ klasörü, {timestamp}-{randomhex}.{ext} dosya adı, JPG/PNG/WebP filter, 2MB limit.
  • middleware/validate.tsvalidateBody(schema) factory — gelen body'yi Zod schema ile parse eder, başarılıysa req.body'ye yazar, hatalıysa ZodError fırlatır (errorHandler yakalar).

schemas/ — body validation şemaları

  • schemas/auth.tsregisterSchema (email + password min 8) ve loginSchema. Email toLowerCase + trim.
  • schemas/resume.tsresumeCreateSchema + resumeUpdateSchema. İçerik z.record(z.unknown()) — şu anda gevşek (TODO: strict).

lib/ — backend helper'ları

  • lib/auth.tshashPassword (bcrypt 10 rounds), verifyPassword, signJwt (1 gün expiry), verifyJwt (payload type guard).
  • lib/prisma.tsPrisma Client singleton. Dev'de globalThis.__prisma cache'i (HMR'de connection pool yeniden açılmasın). @prisma/adapter-pg ile pg driver'ı kullanır.
