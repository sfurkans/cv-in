CV Builder

Modern, hızlı ve özgür bir özgeçmiş oluşturma platformu

→ Yapım Aşamaları

Proje Nedir?

Kullanıcıların form doldurarak profesyonel CV (özgeçmiş) hazırlayabileceği, canlı önizleme ile sonucu anında görebileceği, farklı template'ler arasından seçim yapıp PDF olarak indirebileceği bir web uygulaması.

Kayıt olmaya gerek yok, hassas bilgiler tarayıcıda kalıyor. İsteyen kullanıcı CV'sini public bir link ile paylaşabiliyor. Birden fazla CV oluşturup yönetebiliyor.

Klasik resume builder sitelerinden farkı: gizlilik öncelikli, hızlı ve tamamen ücretsiz.

Hedefler

Kullanıcı için

5 dakikada profesyonel görünümlü, ATS uyumlu bir CV. Login derdi yok, ödeme yok, reklam yok.

Geliştirici (sen) için

Modern frontend (React + TypeScript + Tailwind + Zustand) ve backend (Express + Prisma + PostgreSQL) teknolojilerini gerçek bir projede uçtan uca öğrenmek. Sonunda portfolio'ya koyabileceğin, gerçek kullanıcılara gösterebileceğin bir ürün.

Ürün için

OpenResume, Reactive Resume gibi açık kaynak projelere benzer kalitede, modern tasarımlı ve fonksiyonel bir CV builder.

Tarayıcı (Frontend)

  • Forms: Kullanıcının CV bilgilerini girdiği input ekranları.
  • Zustand Store: Tüm CV verisini tutan global "kasa" — formlar buraya yazar, preview buradan okur.
  • Preview Component: Store'u dinler, veri değişince anında CV önizlemesini yeniden çizer.
  • localStorage: Veriyi tarayıcıda kalıcı saklar — sayfayı kapatsan da kaybolmaz.
  • Axios: Backend'e HTTP istekleri gönderen kütüphane (Phase 9'dan sonra devreye girer).
  • react-pdf: CV verisinden gerçek PDF dosyası üretir, kullanıcıya indirtir.

Sunucu (Backend)

  • Routes: Gelen URL'leri (örn: /api/resumes) ilgili fonksiyonlara yönlendirir.
  • Controllers: HTTP istek/yanıt yönetimi — gelen veriyi alır, yanıtı hazırlar.
  • Services: İş mantığı (business logic) burada — asıl işlemler bu katmanda yapılır.
  • Prisma: JavaScript kodunu SQL sorgularına çeviren ORM — DB ile konuşma çevirmeni.

Veritabanı

  • PostgreSQL: Güçlü, ilişkisel veritabanı sistemi.
  • resumes tablosu: Tüm CV verilerinin JSON formatında saklandığı tek tablo.

Veri Akışı

  1. Kullanıcı bir form alanına yazı yazar (örn: ad soyad, iş deneyimi).
  2. react-hook-form girilen veriyi Zod schema ile valide eder. Hata varsa kullanıcıya gösterir.
  3. Veri geçerliyse Zustand store'a kaydedilir.
  4. Preview component store'u dinler, anında yeniden render olur — kullanıcı CV'nin halini canlı görür.
  5. Aynı anda localStorage'a yazılır (offline backup).
  6. Backend devreye girince (Phase 9), 2 saniye debounced olarak API'ye sync edilir.
  7. Kullanıcı "PDF İndir" derse @react-pdf/renderer veriyi alır, .pdf üretir, dosya inilir.

Önemli Kararlar

Login YOK

Kullanıcı kaydı/girişi eklemiyoruz. Bunun yerine her tarayıcıya bir anonim UUID atanıyor (localStorage'da). Backend bu UUID üzerinden CV'leri sahipleniyor. Sebep: gizlilik öncelikli, hızlı erişim ve auth karmaşıklığından kaçınma.

JSON Resume standardı

Veri modeli için topluluk standardı olan jsonresume.org şemasını baz alıyoruz. Bu sayede başka tool'larla import/export uyumluluğu sağlıyoruz.

Önce frontend, sonra backend

Phase 0-7 tamamen frontend. Backend Phase 8'de geliyor. Bu yaklaşımla frontend'i sağlam temellerle kuruyoruz, sonra entegrasyonu çok daha rahat yapıyoruz. Veri şimdilik localStorage'da, backend gelince oraya senkronlanacak.

PostgreSQL local + JSONB

Geliştirme aşamasında local PostgreSQL kullanıyoruz, deploy aşamasında cloud (Neon, Supabase) kullanacağız. Resume verisini JSONB sütununda tutuyoruz — esnek, schema değişikliği gerektirmiyor.
